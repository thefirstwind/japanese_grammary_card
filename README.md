# 日语高级语法记忆卡片

浏览器内使用的静态页面：打开 **`index.html`**（或通过本地 / GitHub Pages 访问同一目录）即可。语法数据来自 `n1n2语法_复习版_markdown_语音` 等材料的构建结果，当前内置 **287** 张卡片（`cards-data.js`）。

## 三种模式

### 卡片记忆

- 每张卡展示：章节、编号、**JLPT 参考等级（N1 / N2）**、掌握状态、复习次数。
- 内容区块包括：**日文意味**、**中文意思**、**接续**、**用法**、**例文**、**注意**、**学习重点**；部分词条还有系统内嵌的 **惯用法 / 常考常用** 提示。
- **搜索**（仅本模式）：在「词条、日文意思（意味）」中模糊匹配；匹配词在标题等处会高亮。
- **筛选**：章节、状态（未完成 / 不熟悉 / 已掌握）、考级（全部 / 仅 N1 / 仅 N2）。
- **随机抽卡**：在当前筛选结果内打乱顺序浏览一轮。
- **沉浸式**：全屏学习（若浏览器不允许全屏，仍可使用铺满窗口的样式）；**Esc** 或顶部「退出沉浸式」可退出。
- **快捷键**（焦点不在输入框 / 下拉框时）：**←** 上一张，**→** 下一张。
- 可按键标记：**还没完成**、**不熟悉**、**已掌握**；会更新统计区并写入本机 **localStorage**。
- 会记住**上一次停留的卡片**（同一浏览器、同一站点下回打开会尽量恢复）。

### 卡片测试

- 使用上方的 **章节、状态、考级** 筛选出题范围（**不使用**记忆模式里的搜索框）。
- **题量**：10 / 20 / 30 / 全部（在筛选结果内随机抽取）。
- **题型**：
  - 看语法选中文意思  
  - 看语法选日文意思  
  - 看中文意思选语法（会自动跳过没有中文释义的卡片）  
  - 敬语专项：看原型选敬语 / 看敬语选原型（基于第 12 章敬语对应表；若表中无数据会提示无法出题）
- **普通题型**会排除第 12 章编号 **31、35** 两张卡（与总览/敬语表相关）；敬语专项不受此限。
- 每题答毕可看解析（接续、学习重点、惯用法提示、例文等，与普通卡字段一致）。
- **答对**会记入「已掌握」，**答错**会记入「不熟悉」（敬语专项结算文案会提示以对应表为主）。

### 课后练习

- 顶层 **章节 / 类型 / 题组** 与卡片筛选栏相互独立；进入本模式时，上方「章节、状态、考级、搜索」等会隐藏。
- **类型**：各章若有数据可选「课后练习」与「まとめの問題」；若该章没有「まとめ」交互题（如第 12 章），类型下拉会收起，仅保留课后练习相关逻辑。
- **完形（cloze）**：展示整段题目与可展开的 **参考答案**，供自测对答案（不在页内点选判分）。
- **交互题**：「まとめ」等为选择题或带空栏选择，提交后显示对错与小结；可「同一题组再来一轮」。
- 另有「复习版讲义」等打包进来的完形内容（见 `practice-data.js` 生成来源）。

## 统计与进度

- 顶栏统计：**总卡片数**、**未完成**、**不熟悉**、**已掌握**（含占总数的百分比）。
- **重置进度**：清空所有掌握记录（需确认）；**不会**单独清除「记忆位置」，但全部清空后行为以首次进入为准。

进度仅保存在**当前浏览器**，换设备或清空站点数据会丢失。若需备份，可暂时通过开发者工具导出 `localStorage` 中键名 `n1n2GrammarCardProgress.v1` / `n1n2GrammarMemoryPosition.v1`（本界面未提供导入导出按钮）。

## 本地开发与数据重建

- 课后练习等前端数据由脚本生成。若需从原始 Markdown / 文本重新合并 **`practice-data.js`**，在项目目录执行：
  ```bash
  node scripts/build-practice-data.mjs
  ```
- Python 脚本（如 `scripts/extract_review_cloze.py`）与 `scripts/data/` 内素材用于维护数据源；日常学习只需浏览器打开静态文件即可。

## 学习节奏参考（可按需调整）

- **第 1 天**：在记忆模式下过完约一半卡片，把不熟标成「不熟悉」，不必逐条精读。
- **第 2 天**：过完余下一半，再用测试模式针对「未完成」「不熟悉」循环，直到统计区已掌握比例达标。

可参考番茄节奏：每 25 分钟约 25～35 张，休息 5 分钟；几轮之后专刷「不熟悉」。
